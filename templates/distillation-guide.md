# 蒸馏指南 · 仓颉 v2 认知植入术

> 喂进去的是一个人说过的话，提取出来的是他为什么说这些话。

---

## 核心理念

提取的不是"他说了什么"（模型已经知道），而是"他为什么这样想"——这只存在于具体素材中。

**什么值得提取（护城河知识）：**
- 信念的形成故事（"我信X是因为经历了Y"）
- 真实决策 + 反思（决策时的内心考量和事后反思）
- 失败和自我修正（训练数据偏向成功叙事，失败细节最稀缺）
- 内在矛盾（模型倾向让人格一致，真实矛盾最有辨识度）
- 认知边界（真人有明确的能力圈）

**什么不值得提取（会被模型进化吃掉）：**
- 通用观点（"独立思考很重要"——已在训练数据中）
- 显式推理步骤（"Step 1: 分析问题"——模型已内置推理）
- 详细表达规则（"句式偏好短句"——模型越来越擅长风格适配）
- 名言金句（已被引用无数次）

## 收到蒸馏请求后，执行以下 7 步流程

### Step 1：确认对象和素材

**A. 蒸馏谁？**
- 用户明确说了人名 → 直接进行
- 用户丢了文件但没说人名 → 读文件判断，和用户确认

**B. 素材是什么？**
- 用户上传了文件（PDF/TXT/MD/EPUB）→ 直接读取
- 用户只说了人名没给素材 → 用 WebSearch 采集（见下方采集指引）
- 两者都有 → 以用户素材为主，网搜为补充

**C. 聚焦方向？**
- 问用户：全面蒸馏，还是聚焦某个维度（决策方式？创业思维？人生哲学？）
- 用户说"就这样"或不指定 → 默认全面

确认后，创建输出目录：
```
<skill-base>/experts/[person-id]/
├── SKILL.md
└── references/
    └── research/
```

---

### Step 2：消化素材并整理为结构化调研文件

目标不是"读完就提取"，而是**先把原始素材整理成结构化的调研文件，再从调研文件中提取**。

**A. 用户上传了文件**

1. **分块扫描**：长文档先读目录，标记高价值章节，优先精读
2. **带着 7 级清单扫描**：

   | 级别 | 找什么 | 命中后 |
   |:---:|--------|--------|
   | 1 最高 | 信念形成故事（"我信X是因为经历了Y"） | 逐字记录 |
   | 2 高 | 真实决策复盘（做了什么+当时在想什么+事后反思） | 逐字记录 |
   | 3 高 | 失败/错误承认（本人承认的） | 逐字记录 |
   | 4 高 | 内在矛盾暴露（前后不一致、说一套做一套） | 标注两个矛盾点的出处 |
   | 5 中 | 认知边界声明（"这不是我的领域"） | 简要记录 |
   | 6 中 | 思维习惯案例（反复出现的分析模式） | 记录具体案例 |
   | 7 低 | 通用观点/金句 | **没有形成故事就跳过** |

3. **写入提取记录**：写入 `references/research/02-extraction-notes.md`，每条标注优先级、原文出处和提取类型。

**B. 用户没给文件，需要网搜**

按 6 个维度采集，每个维度保存为独立文件：

| 维度 | 存到 | 最低要求 |
|------|------|---------|
| 著作/长文 | `01-writings.md` | ≥2篇长文或1本书核心内容 |
| 访谈/对话 | `02-conversations.md` | ≥1段深度对话 |
| 社交媒体 | `03-expression.md` | ≥20条发言 |
| 外部评价 | `04-external.md` | ≥2个不同视角 |
| 重大决策 | `05-decisions.md` | ≥2个决策+反思 |
| 时间线 | `06-timeline.md` | 关键转折覆盖完整 |

**素材可信度排序**：一手源 > 权威媒体 > 社交媒体 > 二手转述。

**C. 交叉验证（多份素材时执行）**

- 同一信念在 ≥2 份素材中出现 → 标注 ★，置信度高
- 只在 1 份素材中出现 → 保留但标注来源单一
- 不同素材中出现矛盾 → 保留双方，这本身就是内在矛盾的证据

**铁律**：`references/research/` 下最终保留三个文件：
- `01-sources.md`（来源清单）
- `02-extraction-notes.md`（提取记录）
- `03-key-quotes.md`（关键原文引用）

---

### Step 3：提取认知上下文

素材消化完后，按以下结构提取。**只提取模型不可能自己知道的东西。**

#### 3.1 信念及其形成故事（5-7条）

```
每条必须包含：
- 信念本身（一句话）
- 形成故事（什么经历/事件让他开始信这个）
- 关键洞察（这个经历让他意识到了什么）

❌ "独立思考很重要" → 太通用，跳过
✅ "我信独立思考，是因为在YC看了3000个项目后发现最好的想法一开始都被所有人反对" → 有形成故事，保留
```

**铁律：没有形成故事的信念不准进入最终 Skill。**

#### 3.2 真实决策案例（3-5个）

每个决策必须包含：情境（背景）+ 选择（行动）+ 推理（当时在想什么）+ 反思（对了还是错了，学到了什么）

#### 3.3 失败与自我修正（2-3个）

**必须是本人承认过的失败**，不是外部推断的。

#### 3.4 内在矛盾（2-3对）

保留矛盾，不调和。矛盾是人格最有辨识度的特征。

#### 3.5 认知边界（≥3条）

此人明确不懂什么，框架在哪里不适用。

#### 3.6 思维习惯（3-7个）

写成第一人称习惯描述，不是推理步骤：
```
❌ "Step 1: 使用反转思维模型"
✅ "我遇到别人说'这个方案很好'时，本能反应是先想它怎么会失败"
```

#### 3.7 表达 DNA（极简）

只提取 3 条偏好 + 3 条禁忌。模型自己会适配风格。

### Step 4：展示提取摘要，等用户确认

```
蒸馏提取完成，摘要如下：

蒸馏对象：[人名]
素材来源：[N份文件 / N条搜索结果]

提取结果：
- 信念（含形成故事）：N条
- 真实决策案例：N个
- 失败教训：N个
- 内在矛盾：N对
- 认知边界：N条
- 思维习惯：N个

是否继续生成 Skill？
```

用户确认 → Step 5。用户觉得不对 → 回到 Step 3 调整。

---

### Step 5：组装 Skill

读取 `<skill-base>/templates/expert-template.md`，按模板组装。

**四个铁律**：
1. 没有形成故事的信念不进入
2. 没有事后反思的决策不进入
3. 不写 Step 1/2/3 推理步骤
4. 全部第一人称

将 SKILL.md 写入 `<skill-base>/experts/[person-id]/SKILL.md`。

---

### Step 6：质量自检

组装完成后，逐项检查：

- [ ] 每条信念都有形成故事？（不是光秃秃的观点）
- [ ] 每个决策都有事后反思？（不是光秃秃的事实）
- [ ] 失败全部是本人承认的？（不是外部推断）
- [ ] 矛盾是真实存在的？（不是凑数）
- [ ] 认知边界 ≥ 3 条？（具体的"我不懂"声明）
- [ ] 没有通用金句？（"独立思考很重要"这种已删掉）
- [ ] 没有推理步骤？（"Step 1 分析问题"这种已删掉）
- [ ] 表达 DNA ≤ 3+3？（没有详细的句式分析）
- [ ] 素材有保存？（references/research/ 不为空）

全部通过 → 交付。有不通过项 → 回到 Step 3 补充。

---

### Step 7：交付与注册

1. 告诉用户 Skill 已生成及路径
2. **更新专家注册表**：在 `<skill-base>/expert-registry.md` 末尾追加该专家的注册条目：

```yaml
- id: [person-id]
  name: [中文名]
  name_en: [English Name]
  aliases: [别名列表]
  source: cangjie-v2 (自蒸馏)
  domain: [领域列表]
  trigger_keywords: [关键词列表]
  avoid: [回避领域]
  quote: "[代表性语录]"
```

3. 新专家立即可被咨询模式发现和调用。

## 矛盾处理原则

矛盾是人格的核心特征，不是需要修复的 Bug。

| 类型 | 说明 | 处理 |
|------|------|------|
| 时间性矛盾 | 早期说A，后来说B | 记录演化轨迹，以近期观点为主 |
| 领域性矛盾 | 工作中主张X，生活中主张Y | 分领域记录，不强求统一 |
| 本质性张力 | 既追求自由又重视纪律 | 明确记录为核心张力，这是最有意思的部分 |

**错误做法**：❌ 选一边忽略另一边 ❌ 编调和解释 ❌ 假装不存在

## 信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某维度公开信息很少 | 在 Skill 中标注「信息不足，此维度为推测」 |
| 只有二手信息 | 降低置信度，标注为「据[来源]报道」 |
| 信息相互矛盾 | 并列呈现，让用户自行判断 |
| 此人刻意不公开某方面 | 尊重边界，注明「此人对这个话题保持沉默」 |

## 心智模型识别的三重验证

一个论点要被认定为「心智模型」而非「随口一说」：

1. **跨域复现**：同一思维框架出现在至少 2 个不同领域
2. **有生成力**：用这个模型可以推断此人对新问题的可能立场
3. **有排他性**：不是所有聪明人都这样想，体现独特视角

三重验证通过 → 心智模型。仅 1 重 → 降级为决策启发式。0 重 → 不纳入。
